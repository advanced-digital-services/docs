## Importar en Postman

1. Abre Postman
2. **File → Import**
3. Selecciona el archivo `.json` descargado
4. Importa también el zip de environments y elige **Sandbox** para iniciar las pruebas

## Configurar Variables del Environment

El environment espera 3 variables:

| Variable | Descripción | Ejemplo |
|---|---|---|
| `clientId` | OAuth client_id de tu cuenta | `qr-93-550e8400` |
| `clientSecret` | OAuth client_secret | `a1b2c3d4...` |
| `certificatePem` | Certificado X.509 en una sola línea (sin saltos de línea, con `\n` literal) | `-----BEGIN CERTIFICATE-----\n...` |

## Flujo Sugerido

La colección está organizada en el orden de uso típico:

1. **Auth → Generate Token** — ejecuta primero. El response guarda automáticamente el `access_token` en una variable de environment.
2. **Balance → Get Balance** — verifica que el token funcione.
3. **SPEI → Cash-In** — crea un cobro simulado.
4. **Transactions → List** — consulta el estado.

## Carpetas Disponibles

- `Auth` — generación de JWT
- `Balance` — consulta de saldo
- `SPEI` — cash-in, cash-out, transaction by external ID
- `Transactions` — listar
- `Webhooks Config` — listar, crear, eliminar

## Alternativas

¿Prefieres otra herramienta?

- **Insomnia**: importa el mismo `.json` (compatible con Postman v2.1)
- **OpenAPI / Bruno**: usa nuestro [`openapi.json`](/api-reference/openapi.json) directamente
- **cURL**: cada página de endpoint en esta documentación tiene un ejemplo cURL listo para copiar
